Como cada workflow funciona

Clique em cada workflow para ver a explicacao completa do que ele faz, quando dispara e o que acontece em cada etapa.

Trigger: Customer Replied 1. Entrada de Lead — Controle op_aberta ▼

Objetivo: Garantir que cada contato que interage gere exatamente uma oportunidade aberta, sem duplicatas.

Cliente responde→ Verifica tag op_aberta→ Se NAO tem: cria opp→ Adiciona op_aberta→ Round Robin→ Cria tarefa

Se o contato ja tem a tag op_aberta, o workflow encerra sem criar nova oportunidade. Isso evita duplicatas.

Importante: O trigger e "Customer Replied" (resposta do cliente em qualquer canal). NAO dispara ao criar oportunidade manualmente.
Trigger: Opp Status Changed 1c. Controle op_aberta — Remover ao Fechar ▼

Objetivo: Remover a tag op_aberta quando a oportunidade for fechada (Won, Lost ou Abandoned), liberando o contato para gerar nova oportunidade no futuro.

Opp muda para Won/Lost/Abandoned→ Remove tag op_aberta
Atencao: Sem este workflow, um contato que perdeu ou ganhou uma oportunidade nunca mais gerara uma nova automaticamente.
Trigger: Stage Changed 2. SLA Primeiro Contato — 15 minutos ▼

Objetivo: Monitorar se o lead recebido foi contatado dentro de 15 minutos (horario comercial).

Opp entra em Lead Recebido→ Aguarda 15 min (business hours)→ Verifica se ainda esta em Lead Recebido

Se SLA estourado: Atualiza campo SLA para "Estourado", cria tarefa urgente e notifica o gestor.

Se SLA cumprido: Atualiza campo SLA para "Cumprido". O vendedor ja avancou o lead.

Horario comercial: O timer de 15 minutos so conta durante Seg-Sex 08:00-18:00. Se o lead entrar as 17h55, o timer pausa e continua no proximo dia util.
Trigger: Stage Changed 3. Follow-up Proposta Enviada — D+2 ▼

Objetivo: Lembrar o vendedor de fazer follow-up 2 dias uteis apos mover uma oportunidade para "Proposta Enviada".

Opp vai para Proposta Enviada→ Adiciona tag PROPOSTA_ENVIADA→ Aguarda 2 dias uteis→ Verifica se ainda esta la→ Cria tarefa de follow-up

Se a oportunidade ja saiu de Proposta Enviada (avancou ou foi perdida), o workflow encerra sem criar tarefa.

Trigger: Stage Changed 4. Follow-up Negociacao Parada — D+7 ▼

Objetivo: Alertar quando uma oportunidade fica 7 dias sem movimento no stage "Negociacao".

Opp vai para Negociacao→ Aguarda 7 dias uteis→ Verifica se ainda esta la→ Cria tarefa de alerta

A tarefa sugere: verificar objecoes, reforcar proposta de valor, definir prazo ou considerar encerrar a negociacao.

Acao: Cria opp Nutricao 5. Venda Ganha → Pos-Venda ▼

Objetivo: Quando uma venda e fechada (Won), criar oportunidade no pipeline de Nutricao para acompanhamento pos-venda.

Opp status → Won→ Adiciona tag VENDA_GANHA→ Cria opp em Nutricao (Classificacao)
Nota: A opp de Nutricao e criada com nome "Nome - Pos-Venda" e permite duplicadas para clientes que compram mais de uma vez.
Acao: Cria opp Nutricao 6. Perdido/Abandonado → Nutricao ▼

Objetivo: Mover leads perdidos ou abandonados para o pipeline de Nutricao para futura reativacao.

Opp status → Lost ou Abandoned→ Adiciona tag NUTRICAO_ATIVA→ Cria opp em Nutricao (Classificacao)→ Remove tag op_aberta

Lost = lead que rejeitou ou foi rejeitado. Abandoned = lead que sumiu/nao respondeu. A diferenciacao aparece no nome da opp.

Acao: Cria opp Comercial 6c. Requalificado volta ao Comercial ▼

Objetivo: Quando um lead em Nutricao atinge o stage "Requalificado", criar nova oportunidade no pipeline comercial de origem.

Opp Nutricao → Requalificado→ Verifica pipeline de origem (tag)→ Cria opp no pipeline correto→ Tags: REQUALIFICADO + op_aberta→ Remove NUTRICAO_ATIVA
Como funciona: O sistema usa tags de origem (ex: origem_recrutamento) para saber para qual pipeline devolver o lead.
